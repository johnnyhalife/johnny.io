I've been busy so I couldn't write this post early. There's a book that every developer must read; I'm talking about Code Complete 2  written by Steve McConnell. The book is about making better software starting from the way we code it. It's a new classic and every developer should read it once per year. These are some comments about the book:

"Whether you are a new or experienced developer. Code Complete, Second Edition, teaches you the best ways to think about programming." Jeffrey Richter, Author, Applied Microsoft .NET Framework Programming.

"The definitive guide to software construction - the one book a programmer packing for a desert island should take." Diomidis Spinellis, Author, Code Reading: The Open Source Perspective.

"Steve McConnell is one of those rare souls who practice the state of art and demystify it, too" John Vlissides, IBM Research; Coauthor, Design Patterns.

"Steve McConnell knows more ab out building software than anyone else; we're lucky he has packed all his insight and practical experience into this important original book". Alan Cooper, "Father of Visual Basic"; Author, About Face.

I'll be posting more books I think you should read or books that I've read and enjoyed.
